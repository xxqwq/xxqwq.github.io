网页的组成部分：

  • html:网页的结构
  • CSS：网页的样式（美化）
  • JavaScript:网页的交互（灵魂）

浏览器和浏览器内核

浏览器内核

  • 浏览器的最核心的部分：渲染引擎，也称为浏览器内核

  • 常见的浏览器内核：

    1. Trident(三叉戟):IE,360,搜狗，百度，UC

    2. Gecko(壁虎)：Mozilla,Firefox

    3. Presto(急板乐曲)->Blink(眨眼):Opera

    4. Webkit:Safari、360、搜狗、移动端

    5. Webkit-?Blink:Google Chrome,Edge

      • 不同的浏览器内核有不同的解析、渲染规则，所以同一网页在不同内核的浏览器中的渲染效果也可能不同

      • 浏览器内核读取代码并且按代码渲染出来

HTML(超文本标记语言)

  • 是一种用于创建网页的标记语言（不属于编程语言,属于计算机语言）

    • 有标签和内容组成的部分称为元素

  • HTML元素是构建网站的基石

    特点：

    1. 后缀：.html

文档声明

`

  • 文档声明:HTML最上方的一段文本，声明文档类型

  • HTML文档声明,告诉浏览器当前页面是HTML5页面

  • 让浏览器用HTML5的标准去解析识别内容

  • 必须放在HTML文档最前面，不能省略，省略了会出现兼容性问题

html元素

元素表示一个HTML文档的根（顶级元素），又称**根元素**
  • 其他元素必须是这个元素的后代
  • W3C标准建议为html元素增加一个lang属性
    • 帮助语音合成工具确定要使用的发音
    • 帮助翻译工具确定要使用的翻译规则

      • lang=”zh-CN”表示中文

img元素：将一份图像嵌入文档

相对路径和图片格式

相对路径（relative）:相当于当前文件的一个路径

  • 路径分隔符统一用**/**

  • 1个” .”代表当前文件夹，可省略

  • 2个”.”代表上级文件夹

div（division 分开）元素和span (跨域、涵盖)元素

  • 都是纯粹的容器用来包裹内容

区别：

  • div：多个div元素包裹的 内容会在不同行显示

    • 一般作为其他元素的父容器把其他元素包住代表一个整体
    • 用于把网页分割为多个独立的部分

  • span：多个span元素包裹的内容会在同一行显示

    • 默认情况下和普通文本没区别
    • 用于区分特殊文本和普通文本，比如用来显示一些关键字

不常用元素:

  • strong:内容加粗、强调

    • 通常加粗用css

  • i：内容倾斜

  • code:用于显示代码

    • 偶尔用来显示等宽字体

  • br:换行元素

HTML常见的全局属性:

  • id:定义唯一标识符，该标识符在整个页面中必须是唯一的，目的是在链接(使用片段标识符)、脚本或样式（css）时标识元素

  • class:一个以空格分隔的元素的类名（classes）列表，它允许CSS和JavaScript通过类选择器或者DOM方法来选择和访问特定的元素

  • style:给元素添加内联样式

  • title:包含表示与其所属元素相关信息的文本，这些信息通常可以作为提示呈现给用户，但不是必须的

字符实体

  • HTML实体是一段以**连字号(&)**开头，以分号(;)结尾的文本 —&名称/代码；
  • 实体常常用于显示保留字符(会被解析为HTML代码的)和不可见的符号(比如空格)
  • 也可以用实体代替其他难以用标准键盘键入的字符

SEO

  • 搜索引擎优化（英语：search engine optimization，缩写为SEO）是通过了解搜索引擎的运作规则来调整网站，以及提高网站 在有关搜索引擎内排名的方式。

CSS：层叠样式表，又称为又称串样式列表、级联样式表、串接样式表、阶层式样式表） 是为网页添加样式的代码。

  • css不是编程语言，也不是标记语言，是一门样式表语言

  • CSS的出现是为了美化HTML的，并且让结构（HTML）与样式（CSS）分离；

    • 美化方式一：为HTML添加各种各样的样式，比如颜色、字体、大小、下划线等等；

    • 美化方式二：对HTML进行布局，按照某种结构显示（CSS进行布局 – 浮动、flex、grid）；

规则：

  • 声明（Declaration）一个单独的CSS规则，如 color: red; 用来指定添加的CSS样式。
    • 属性名（Property name）：要添加的css规则的名称；
    • 属性值（Property value）：要添加的css规则的值；

      • RGB颜色可以通过以#为前缀的十六进制字符和函数（rgb()、rgba()）标记表示。
      • 方式一：十六进制符号：#RRGGBB[AA]
        • R（红）、G（绿）、B （蓝）和A （alpha）是十六进制字符（0–9、A–F）；A是可选的。
      • 方式二：十六进制符号：#RGB[A]
        • R（红）、G（绿）、B （蓝）和A （alpha）是十六进制字符（0–9、A–F）；
        • 三位数符号（#RGB）是六位数形式（#RRGGBB）的减缩版。
        • 四位数符号（#RGBA）是八位数形式（#RRGGBBAA）的减缩版。
      • 方式三：函数符： rgb[a](R, G, B[, A])
        • R（红）、G（绿）、B （蓝）可以是（数字），或者（百分比），255相当于100%。
        • A（alpha）可以是0到1之间的数字，或者百分比，数字1相当于100%（完全不透明）。

CSS文本属性

text-decoration(常用)

  • 用于设置文字的装饰线
  • 常见取值:
    • none:无装饰线，可去除a元素默认的下划线
    • underline:下划线
    • overline:上划线
    • line-torough:中划线（删除线）

text-transform-indent

  • transform用于设置文字的大小写转换,indent用于缩进

  • 常见值:

    • capitalize:：将每个单词的变为大写
    • uppercase：将每个单词的所有字符变为大写
    • lowercase：将每个单词的所有字符变为小写
    • none:无影响

  • text-indent与em(相对于字体的大小)结合

    • 例:text-indent: 2em;

text-align(重要)

  • 直接翻译过来设置文本的对齐方式
  • 定义行内内容(例如文本)如何相对他的块父元素对齐
  • 作用对象只能是对行内级元素，类似于div的是块级元素，但可以修改display:inline-block,或者用margin:0 auto
  • 常用的值:
    • left:左对齐
    • right:右对齐
    • center：正中间对齐
    • justify：两端对齐
      • 一般最后一行无效

letter-spacing、word-spacing

  • 用于设置字母、单词之间的间距
  • 默认是0，可以设置为负数

字体属性

font-size

  • 决定文字的大小

  • 常用设置方式：

    • 具体数值加单位
    • 百分比：
      • 基于父元素的font-size计算

  • 可以被继承

font-family(重要，不过一般设置一次)

  • 可以被继承

  • 用于设置文字的字体名称

  • 可以设置一个或多个字体名称

  • 浏览器会选择列表中第一个该计算机上有安装的字体

  • 或者是通过**@font-face指定的可以直接下载的字体**

font-weight(重要)

  • 用于设置文字的粗细
  • 常见的取值:
    • 100|200|300….
    • normal:400
    • bold:700

font-style(一般)

  • 用于设置文字的常规、斜体显示
  • 常见的取值:
    • normal:常规显示
    • italic(斜体)：用字体本身设计的的斜体显示（通常有专门的字体）
    • oblique(倾斜)：文本倾斜显示（仅仅是文字）所有的都是一样的样式

font-variant（一般）

  • 可以影响小写字母的显示形式

  • 常见取值:

    • normal：常规显示
    • small-caps:将小写字母替换为缩小后的大写字母

line-height（常用）

  • 用于设置文本的行高

    • 理解为一行文字所占据的高度

  • 行高由文字及两端空白组成，严格定义是:两行文字基线之间的间距

    • 基线：与小写字母x最底部对着的线
    • 行距：上一行底线与下一行顶线之间的距离

  • 应用:假设div中只有一行文字，如何让他垂直居中

    • 让line-height=height

font

  • font是一个缩写属性，可以用来作为font-style,font-family等的简写

  • font-style font-variant font-weight font-size/line-height font-family

  • 规则:

    • 前三个可以随意调换顺序，也可以省略

    • /line-height可以省略,不省略必须在size后面

    • font-size、font-family不可以调换顺序，bu’ke’y

      • 一般做一些通用的设置

    • 类选择器

    • id选择器

简单选择器（重要）

  • 元素选择器，使用元素的名称
  • 类选择器，使用**.类名**
  • id选择器，使用#id
    • 注意事项:
      • 一个HTML里面的id值是唯一的，不可重复
      • id如果有多个单词组成，单词之间可以用中划线下划线链接，也可以使用驼峰表示
      • 最好不要用标签名作为id值
      • 中划线又叫做连字符

属性选择器

  • 拥有某一个属性[属性]{}
  • 属性包含某个值:[att=val]

后代选择器（重要）

  • 一：所有的后代（直接，间接）

    • 后代之间用空格分隔

  • 二：直接子代选择器（必须是直接子代）

    • 选择器之间以**>分割**

兄弟选择器

  • 相邻兄弟选择器
    • 使用+连接
  • 全兄弟选择器
    • 使用~连接

选择器组-交集选择器

  • 交集选择器：需要同时符合两个选择器条件（两个选择器要紧密连接）

  • 并集选择器：符合一个选择器条件即可（两个选择器用**,**分割)

伪类（Pseudo-classes）

  • 伪类是选择器的一种，用于选择处于特定状态的元素

    • 特定状态比如说鼠标放上去

  • 常见的伪类:

    1.动态伪类

    • :link 、:visited 、:hover(悬停) 、:active、 :focus
    • a:link 未访问的链接
    • a:visited:已访问的链接
    • a:hover:鼠标挪动到连接上（重要）
    • a:active 激活的链接（鼠标在连接上长按住未松开）
    • 注意
      • hover必须放在link和visited后面才能完全生效
      • active必须放在hover后面才能完全生效
      • 顺序建议为：link、visited、hover、active
      • hover、link也能用于别的元素

​ 2.目标伪类:target

​ 3.语言伪类:lang()

​ 4. 元素状态伪类:enabled、 :disabled 、：checked

​ 5.结构伪类

​ 6.否定伪类:not()

  • 所有的伪类: https://developer.mozilla.org/zh-CN/docs/Web/CSS/Pseudo-classes

伪元素（重要）

  • 可以写一个冒号也可以写两个，推荐写两个

CSS属性的特性

继承（inherited）

  • 如果一个属性具备继承性，那么在该元素上设置后，他的后代元素都可以继承这个属性

  • 当然如果后代元素自己有设置该属性，那么优先使用后代元素自己的属性

  • 如font-size/font-family/font-weight/line-height-color/text-align都有继承性

  • 常见的继承：

    • color
    • cursor
    • font
    • letter-spacing
    • line-height

  • 继承过来的是计算值，不是设置值

  • 强制继承-属性名:inherit(很少用)

层叠

  • 对于一个元素来说，向同一个属性可以通过不同的选择器给他进行多次设置
  • 那么属性会被一层层覆盖上去
  • 但是最终只有一个会生效
  • 判断方法:
    • 方法一：选择器的权重，权重大的生效
      • !important：1000—>color:red !important
      • 内联样式:1000
      • id选择器:100
      • 类选择器、属性选择器、伪类:10
      • 元素选择器、伪元素:1
      • 通配符:0
    • 方法二：先后顺序，权重相同时，后面设置的生效

HTML元素的类型

  • 块级元素(block-level elements):独占父元素的一行，可以设置宽度和高度
  • 行内级元素(inline-level elements):多个行内级元素可以在父元素的同一行中显示，（非替换元素）不可以设置宽度和高度，是由内容决定
    • img元素是行内替换元素
    • 行内替换元素:
      • 和其他的行内级元素在同一行显示
      • 可以设置宽度和高度
      • p元素不要包裹div元素–》行内级元素不要放块级元素

display

  • 常用属性:
    • block:显示为块级元素
    • inline:显示为行内级元素
    • inline-block：同时具备行内级、块级元素的特征，可以和别的元素在同一行显示，也可以设置宽度和高度
    • none:隐藏元素

元素隐藏方法

  • 方法一：display设置为none

    • 元素不显示出来，不占据位置和空间

  • 方法二：visibility设置为hidden

    • 元素不可见但是会占据空间
    • 默认为visible

  • 方法三：rgba设置颜色，将a设置为0（a取值为0~1）

    • a设置的是alpha值，设置透明度，不影响子元素

  • 方法四：通过背景颜色透明度来隐藏

    • background-color:transparent(只设置背景)
    • opacity：设置透明度，并且会携带所有的子元素都有一定的透明度

元素的内容溢出-overflow

  • 用于控制内容溢出时的行为
  • visible:溢出的内容依然可见
  • hidden:溢出的内容直接裁剪
  • scroll:溢出的内容被裁减，但通过滚动机制查看，滚动条占用的空间属于width、height
  • auto:自动设置

外边距margin

  • 用于设置盒子的外边距，通常用于元素和元素之间的间距

  • 包括四个方向

    • top(相对于父元素的宽度)
    • right
    • bottom
    • left

  • 水平方向不折叠，竖直方向折叠

  • 父子关系最好用padding

  • 上下传递问题

    • 如果块级元素的顶部线限和父元素的顶部线重叠，那么他的margin-top会传递给父元素
    • 如果块级元素的底部线与父元素的底部线重叠，并且父元素的高度是auto，那么他的marigin-bottom会传递给父元素
    • 如何防止
      • 给父元素设置padding-top/bottom
      • 给父元素设置border
      • 触发BFC：设置overflow为auto

  • 上下折叠问题

    • 垂直方向上相邻的两个margin可能会合并为一个margin，叫做折叠(collapse)
    • 两种情况:
      • 两个兄弟块级元素之间的上下折叠
      • 父子块级元素之间的折叠
    • 计算规则：
      • 两个值比较，去较大的值

  • padding一般用来设置父子元素之间的间距

  • margin一般用来设置兄弟元素之间的间距

布局

标准流(NORMAL FLOW)

  • 默认情况下，元素都是按照normal flow（标准流、常规刘、正常流、文档流）进行排布

    • 从左到右、从上到下按顺序摆放好
    • 默认情况下，互相之间不存在层叠现象

  • margin-padding位置调整

    • margin还可以设置负数
    • 缺点是设置一个元素的marigin和padding会影响到其他元素的定位
    • 不便于实现层叠的效果

  • 如果想要元素跳出标准流单独定位可以通过position来设置

    • fixed

元素的定位

  • 定位允许从正常的文档流布局中去除元素，使他们具有不同的行为

  • position可以对元素进行定位，常见的取值有:

    • relative、absolute、sticky、fixed让元素变成定位元素

      • static：默认值 静态定位

        • 元素按照normal flow布局
        • left、right、top、bottom没有任何作用

      • relative：相对定位

        • 元素按照normal flow布局
        • 可以通过left、right、top、bottom进行定位
        • 定位参照对象是元素自己原来的定位
        • 应用场景：在不影响其他元素位置的情况下，对当前元素位置进行微调

      • fixed：固定定位、

        • 元素脱离标准流
        • 可以通过left、right、top、bottom进行定位
        • 定位参照对象是视口（viewport）
        • 当画布滚动时，固定不动

      • absolute：绝对定位

        • 元素脱离标准流
        • 可以通过left、right、top、bottom进行定位
        • 定位参照对象是最邻近的定位祖先元素
        • 如果找不到这样的祖先元素，参照对象是视口
        • 子绝父相
          • 在绝大数情况下，子元素的绝对定位都是相对于父元素进行定位
          • 如果希望子元素相对于父元素进行定位，又不希望父元素脱标，常用解决方案是：
            • 父元素设置position: relative（让父元素成为定位元素，而且父元素不脱离标准流） 
            • 子元素设置position: absolute
            • 简称为“子绝父相”

      • sticky：粘性定位

        • 可以看做是相对和固定（绝对）定位的结合体
        • 它允许被定位的元素表现得像相对定位一样，知道滚动到某个阈值点
        • 当达到这个阈值点，就会变成固定（绝对定位），比如top:0 ;
        • sticky是相对于最近的滚动祖先包含滚动视口的(the nearest ancestor scroll container’s scrollport

  • z-index

    • 用来设置定位元素的层叠顺序（仅对定位元素生效)
    • 取值可以使正整数、负整数、0
    • 比较原则：
      • 如果是兄弟
        • z-index越大，层叠再越上面
        • z-index相等，写在后面的那个元素层叠在上面
      • 如果不是兄弟
        • 各自从元素自己以及祖先元素中，找到最邻近的两个定位元素进行比较（同一层）
        • 而且这两 个定位元素必须有设置z-index的具体数值

元素的浮动

  • float属性可以指定一个元素应沿着容器的左侧或右侧放置，允许文本和内联元素环绕它

    • float 属性最初只用在一段文本内浮动图像，实现文字环绕
    • 后来也用于左右布局

  • 绝对定位、浮动都会让元素脱离标准流

  • float属性：

    • none：默认值
    • left：向左浮动
    • right：向右浮动

  • 浮动规则：

  • 去除多个行内级元素的中间的空格

    1. 删除换行符
    2. 将父级元素的font-size设置为0，但是需要子元素设置回来
    3. 子元素全部统一向同一个方向浮动
    4. flex布局

浮动的问题：高度塌陷

  • 由于浮动元素脱离了标准流，不再像父元素汇报高度
  • 父元素计算总高度时，就不会计算浮动子元素的高度，导致了高度坍塌
  • 解决问题：清浮动
    • 目的是让父元素计算总高度是，把浮动子元素的高度算进去
    • 1.使用clear属性
    • 给父元素增加：：after伪元素
    • 2.给父元素设置固定高度
      • 扩展性不好
    • 3.在父元素最后增加一个空的块元素，并且设置clear：both
      • 会增加很多无意义的空标签
      • 违背了结构与样式分离的原则

clear属性

  • 可以指定一个元素是否必须移动 （清除浮动后）到在他之前的浮动元素下面

  • 常用取值：

    • left：要求元素的顶部低于之前生成的所有左浮动元素的底部
    • right：要求元素的顶部低于之前生成的所有右浮动元素的底部
    • both：要求元素的顶部低于之前生成的所有浮动元素的底部
    • none：默认值

  • 直接子元素叫flex item

    • 特点：
      • flexitem 的布局将受lex container属性的设置来进行控制和布局
      • flex item不在严格区分块级元素和行内级元素
      • flex item默认情况是包裹内容的，但是可以设置宽度和高度

  • 设置display为flex或者inline-flex

    • flex:flex container以block-level形式存在
    • inline-flex：flex container以inline-level形式存在

flex-direction

  • flex items 默认都是沿着 main axis（主轴）从 main start 开始往 main end 方向排布
    • flex-direction 决定了 main axis 的方向，有 4 个取值
    • row（默认值）、row-reverse、column、column-reverse

  • 取值：

    • wrap：多行
    • wrap-reverse：多行，cross反过来

  • flex-flow是direction和wrap的简写，顺讯随意，都可以省略

transform（形变）属性

  • 允许对某一个元素进行某些形变，包括旋转、缩放、倾斜或平移

  • 并非所有的盒子都可以转换（通常行内级元素不能形变）

  • transform-origin可以修改形变原点位置

  • 常见的transform function有

    • 平移：translate(x,y)
      • 用于移动元素在平面上的位置
      • 一个值时，设置x轴上的位移
      • 两个值时：设置y轴上的位移
      • 百分比是相对于自身的
      • 是translateX（）和translateY（）的缩写
    • 缩放：scale(x,y)
      • 可改变元素的大小
      • 一个值时，设置x轴上的缩放
      • 两个值时：设置y轴上的缩放
      • 可以设置数字也可以设置百分比
    • 旋转：rotate(deg)
      • 一个值：表示旋转的角度
      • 值类型：
        • deg
        • 正数为顺时针
        • 负数为逆时针
    • 倾斜：skew(deg,deg)
      • 一个值表示x轴上的倾斜
      • 两个值时，表示x轴和y轴上的倾斜
      • 值类型：
        • deg：旋转的角度
        • 正数为顺时针
        • 负数为逆时针
      • 倾斜原点搜transform-origin 的影响

  • 设置多个值：translate、scale、skew、rotate

  1. top/translate

  • 让元素向上位移自身的50%，translateY(50%)

transition(过渡)动画

◼ transition CSS 属性是 transition-property，transition-duration，transition-timing-function 和 transition-delay 的 一个简写属性。

◼ transition-property：指定应用过渡属性的名称 ：可以写all表示所有可动画的属性  属性 是否支持动画查看文档

◼ transition-duration：指定过渡动画所需的时间 ，单位可以是秒（s）或毫秒（ms）

◼ transition-timing-function：指定动画的变化曲线 https://developer.mozilla.org/zh-CN/docs/Web/CSS/transition-timing-function

◼ transition-delay：指定过渡动画执行之前的等待时间
